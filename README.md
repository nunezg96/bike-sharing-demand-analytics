# Bike Sharing Demand Analytics

## Project Overview

This project presents an end-to-end analysis of 12 months of bike-sharing data containing millions of ride records. The goal was to identify ridership trends, seasonal demand, and usage patterns that could support data-driven operational and customer engagement decisions.

The analysis included data cleaning, transformation, feature engineering, exploratory data analysis, and the development of an interactive Tableau dashboard.

## Business Problem

How can historical bike-sharing data be used to identify demand patterns, understand rider behavior, and support operational planning and customer engagement strategies?

## Objective

Analyze historical bike-sharing data to:

- Identify seasonal demand patterns
- Explore changes in ridership over time
- Identify peak usage periods
- Analyze recurring usage trends
- Communicate actionable insights through an interactive dashboard

## Tools & Technologies

- Python
- Pandas
- R
- dplyr
- tidyverse
- Tableau
- Matplotlib

## Data Preparation

The analysis used 12 months of bike-sharing data containing millions of ride records.

The data preparation process included:

- Consolidating monthly datasets into a unified analytical dataset
- Reviewing missing values and duplicate records
- Validating ride duration and removing invalid observations
- Transforming date and time fields
- Creating analytical features for further exploration

New features included:

- Ride duration
- Day of week
- Month
- Year
- Hour
- Season
- Time-of-day classifications

## Exploratory Data Analysis

Exploratory data analysis was performed to identify patterns in bike-sharing activity across different time periods.

The analysis focused on:

- Seasonal ridership trends
- Monthly changes in demand
- Daily usage patterns
- Peak riding periods
- Changes in activity by time of day

## Tableau Dashboard

An interactive Tableau dashboard was developed to communicate key ridership trends and usage patterns.

The dashboard allows users to explore historical bike-sharing activity and identify periods of high and low demand.

## Key Insights

- Bike-sharing activity demonstrated clear seasonal demand patterns.
- Ridership changed significantly throughout the year.
- Usage patterns varied across different times of day.
- Peak riding periods revealed recurring demand trends.
- Historical ridership patterns can support operational planning and customer engagement decisions.

## Business Recommendations

- Use historical demand patterns to support bike availability and operational planning.
- Align customer engagement strategies with periods of changing seasonal demand.
- Use peak usage trends to identify opportunities for targeted promotions.
- Continue monitoring seasonal and time-based usage patterns to support service optimization.
